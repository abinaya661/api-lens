# Moonshot Setup Guide for API Lens

## Current API Lens status

Not ready in the current key-based tracking flow.

Moonshot keys can be created and used for inference, but we do not currently have a confirmed public usage or billing API that fits the current API Lens product promise.

## What the user can obtain today

- A normal Moonshot API key from the Moonshot open platform

## Step-by-step

1. Sign in to the Kimi or Moonshot open platform.
2. Open the `API Key Management` section.
3. Click `New` to create a new API key.
4. Copy the key immediately and store it safely.

## Important note for API Lens

Do not market Moonshot as a working add-key provider yet.

Why:

- API Lens can validate that the key is real for inference.
- We do not currently have the provider-side billing or usage endpoint needed to support the "trackable or reject" rule with confidence.

## What to tell the user

Moonshot key creation is straightforward, but API Lens should not accept Moonshot as a finished billing-tracking integration yet.
