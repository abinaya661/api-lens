1. Executive Summary
The API Lens codebase has a solid UI shell across 8 dashboard pages, working Supabase Auth, security headers, Sentry instrumentation, and a CI pipeline. Server actions, TanStack Query hooks, and Resend email integration have been added.
Several critical systems remain incomplete: the Trigger.dev sync pipeline is stub code, rate limiting is defined but not enforced, RLS policies are overly permissive, 5 provider adapters are missing, and the payment system needs to be rewired from Razorpay to Dodo Payments.
Payment provider has been changed from Razorpay to Dodo Payments, which acts as a Merchant of Record. This eliminates the need for a multi-provider setup (Razorpay + Paddle) and handles global tax compliance (VAT, GST, sales tax) across 190+ countries automatically. Dodo also provides native discount/promo code management via API and dashboard.

6. Deployment Platform Recommendation
Given API Lens is a Next.js 15 app with Supabase, Trigger.dev, and Dodo Payments (MoR), targeting 10K–100K users:
Phase 1 (MVP Launch, 0–10K users): Vercel
Zero-config Next.js deployment, global CDN, built-in cron support, and the fastest path to production. At 10K users, costs stay at $60–150/month on Pro plan ($20/seat/month). Dodo Payments is provider-agnostic and works from any hosting platform.
Phase 2 (Growth, 10K–100K users): Vercel + Railway Hybrid
Keep Next.js frontend on Vercel. Move Trigger.dev background processing (sync pipeline, heavy provider API calls) to Railway for predictable container-based pricing. Total: $200–600/month for 100K users.
Why Dodo simplifies hosting decisions:
Unlike the old Razorpay + Paddle split which required location detection and routing, Dodo is a single provider that works everywhere. No Vercel geolocation headers needed, no multi-webhook complexity. One integration works for India and international users identically.
